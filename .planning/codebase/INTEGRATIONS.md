## APIs & External Services

**Backend API:**
- Custom REST backend - Provides auth, user profile, blog posts, and file uploads
  - SDK/Client: Custom `API` wrapper in `src/api/index.ts` (native `fetch`, `credentials: 'include'`)
  - Auth: Cookie-based (`access`, `refresh` HttpOnly; `has_session` JS-readable)
  - All calls go to relative `/api/â€¦` â€” Next.js rewrites proxy to `API_URL`
  - Endpoints used: `POST /auth/login`, `POST /auth/logout`, `POST /auth/refresh`, `GET /profile`, `GET /posts`, `GET /posts/:slug`

**Google Analytics:**
- GA4 (Google Tag Manager) - Page view tracking
  - Script: `https://www.googletagmanager.com/gtag/js?id=${GA_ID}` loaded via `next/script` with `strategy="afterInteractive"`
  - Auth: `GA_ID` environment variable (if unset, scripts are skipped)
  - Configured in: `src/app/layout.tsx`

**Google Fonts:**
- Roboto font - loaded via `next/font/google` (self-hosted by Next.js at build time)
  - Configured in: `src/app/layout.tsx`
  - Weights: 400, 700; subsets: latin

## Data Storage

**Databases:**
- None client-side â€” all persistence is delegated to the backend
  - Connection: `API_URL` env var (`http://localhost:4000` default)
  - Client: `src/api/index.ts` (`API` object)

**File Storage:**
- Backend-managed file uploads; URLs returned from `/uploads/:path*`
- Upload URLs are proxied through Next.js via the `/uploads/:path*` rewrite in `next.config.ts`
- `src/shared/lib/uploads.ts` provides `toProxiedUrl()` to convert absolute backend URLs to root-relative paths

**Caching:**

## Authentication & Identity

**Auth Provider:**
- Custom backend (no third-party auth provider)
  - Implementation: Cookie-based session; backend sets `access` (HttpOnly), `refresh` (HttpOnly), `has_session` (JS-readable) on login/refresh
  - FE auth logic: `src/shared/lib/cookies.ts`, `src/shared/contexts/UserContext.tsx`, `src/api/auth.ts`
  - Token refresh: `src/api/index.ts` automatically retries on 401 if `has_session` cookie is present (single retry via `POST /auth/refresh`)
  - Route guard: `src/proxy.ts` (Next.js 16 middleware) gates `/blog/cover` behind session cookie presence
  - JS cannot read or delete `access`/`refresh` â€” they are HttpOnly; only `has_session` is observable client-side

## Environment Configuration

**Required env vars:**
- `API_URL` - Backend origin for `/api` and `/uploads` rewrite (default: `http://localhost:4000`); set as Docker build ARG and runtime env
- `GA_ID` - Google Analytics measurement ID (optional; omit to disable analytics)

**Compose-only env vars:**
- `FRONTEND_HOST` - Virtual host for nginx-proxy
- `LETSENCRYPT_EMAIL` - Email for Let's Encrypt cert registration
- `NETWORKS_DRIVER` - Docker network driver
- `RESTART_POLICY` - Docker restart policy
- `BASIC_AUTH`, `BASIC_AUTH_USERNAME`, `BASIC_AUTH_PASSWORD` - Optional nginx basic auth
- `COMPOSE_PROJECT_NAME` - Used for container hostnames

**Secrets location:**
- `.env.local` (development, gitignored) and `.env.production` (production, gitignored)
- `.env.example` is committed and documents all variables
